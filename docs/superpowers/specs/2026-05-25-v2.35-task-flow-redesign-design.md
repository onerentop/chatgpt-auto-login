# v2.35 — 任务流驱动的前端重设计（设计 spec）

> 日期：2026-05-25  
> 范围：v2.34 完成了 design token + 包壳重排。本轮**按"运营每天做什么"
> 而不是"后端实体"重新组织前端**：Dashboard 改工作台 hub、加 Command
> Palette / Pipeline HUD、Accounts toolbar 上下文操作栏、视觉色板更
> 现代。保留所有现有功能 / API / 后端逻辑。

## 一、为什么 v2.34 还不够

v2.34 达到了"简约 + 一致"。但用户反馈"要符合人类操作、分析每个功能
应该对应人类的操作"，这是 jobs-to-be-done 维度，不是视觉维度。当前
IA 仍按后端实体分页面（Accounts / Execute / Results / Config /
Dashboard），实际运营 flow 是任务驱动的：

- **晨间巡检**：昨晚跑了多少？多少失败需要处理？代理还活着没？
- **导入新批次**：批量导入 50-200 个邮箱。
- **跑一批新账号**：选号 → 启动 → 监控 → 看结果。
- **重试失败**：按失败原因分类，分别处理。
- **导出凭证**：下载 JSON 给下游使用。
- **测活老账号**：定期检查 Plus 还活着否。

这些任务里，当前 Dashboard 只是"统计页面"，没有"今天我要做什么"的入
口；Execute 跑中时缺一个全局可见的 Pipeline 状态；Accounts toolbar
2 行所有 filter 始终展开虽然密度提升但仍嘈杂；没有 Command Palette
让重度用户跳过 navigation 直接到操作。

## 二、本轮范围

### 2.1 做（6 件大事）

1. **现代视觉系统升级**（B1）—— 在 v2.34 token 之上把主色 / 阴影 /
   圆角 / 微动效推到 Linear / Vercel / Notion 水准。
2. **Dashboard 改工作台**（B2）—— Hero metrics + 任务卡片 + 系统健
   康 + 近期活动流。
3. **Command Palette (Ctrl/Cmd+K)**（B3）—— 全局快捷操作面板，覆盖
   导航 / 账号 / 执行 / 配置 / 主题 5 组。
4. **Pipeline HUD**（B4）—— Execute 跑中时顶部出现实时进度带（当前
   账号 + phase + X/Y + ETA + 主代理节点）。
5. **Accounts toolbar 上下文操作栏**（B5）—— 筛选收 1 行（核心字段 +
   "更多筛选" popover 折叠次要项）；选中操作改 sticky context bar
   从底部 slide-in 仅在选中时显示。
6. **EmptyState 升级 + 微动效**（B6）—— 简单 SVG / CSS 插画；hover /
   focus 全局过渡；toast 圆角阴影。

### 2.2 不做（明确排除）

- 后端 / API / socket 协议改动。
- status.js 单一来源；selection.js；4 个 composable 的语义。
- 路由结构（5 视图保留；不合并 / 不拆分）。
- 引入新 npm 依赖 — 全部用 Element Plus + 已有 icons + 原生 CSS。
- 国际化、移动端响应式专项。
- 把 el-tag 替换为自定义 StatusBadge —— 涉及面太广，留 v3.0。
- 真后端 metrics / ETA 算法（Pipeline HUD 的 ETA 用前端启发式：平
  均最近 5 个账号耗时）。

### 3.2 B2 Dashboard 改工作台 Hub

替换当前 7 KPI grid 为 4 区结构：

```
┌────────────────────────────────────────────────────────┐
│ PageHeader: 仪表盘 — 今日 2026-05-25 周一              │
├────────────────────────────────────────────────────────┤
│ Hero metrics (4 张大卡):                                │
│ ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐            │
│ │ 总账号 │ │ 今日成 │ │ 待重试 │ │ 运行中 │            │
│ │  142   │ │  ▲ 23  │ │  ▼ 5   │ │   3    │            │
│ │ +12 周 │ │ 上周 18│ │ -2 时前│ │  跑中  │            │
│ └────────┘ └────────┘ └────────┘ └────────┘            │
├──────────────────────────────┬─────────────────────────┤
│ 任务卡片（左 2/3 区）:        │ 系统健康（右 1/3）：    │
│ [➕ 导入新账号]              │ ● 引擎     idle         │
│ [▶ 跑一批新账号]             │ ● 主代理   15/15 alive  │
│ [🔄 重试失败 (5)]            │ ● JP 通道  1/1 alive    │
│ [💚 测活老账号]              │ ● WebSocket connected   │
│ [⬇ 下载本批凭证]              │ ● uptime   2h 14m       │
│                              │ [/api/health 详情]       │
├──────────────────────────────┴─────────────────────────┤
│ 近期活动（按时间倒序）：                                │
│ • 02:30  user@xxx 激活 Plus 成功                       │
│ • 02:28  代理切换到 pro-美国03（pro-美国01 失败 3 次）  │
│ • 02:25  user@yyy verify_error: $20.0 not free         │
│ • ... 翻 15 条                                          │
└────────────────────────────────────────────────────────┘
```

数据源：
- **Hero metrics**：今日成 = `results.filter(r => isPlus(r.status) && r.updated_at startsWith today)`；待重试 = `isFailedToRetry`；运行中 = `r.status === 'running'`。+12 周 / 上周 18 等同环比，用 `updated_at` 切今日 / 昨日 / 上周。
- **任务卡片**：每个卡片是大按钮 (icon + 标题 + 副标 + 跳转)，点击直接跳对应视图带 query。例：[导入新账号] → `/accounts?action=import`（视图 mount 时若 query.action='import' 自动打开导入弹窗）。
- **系统健康**：`GET /api/health`（v2.30 HX-16 已有）；轮询 10s。
- **活动流**：`GET /api/liveness/logs?limit=15`（已有）+ 后端 socket 实时新增。

### 3.3 B3 Command Palette (Ctrl/Cmd+K)

新组件 `web/src/components/ui/CommandPalette.vue`：

- 全局键盘监听 Ctrl/Cmd+K → 打开 modal（fixed top center，宽 600px）。
- 顶部一个 large input + 下方过滤后的 command 列表（分组）。
- 命令分组（硬编码 array，每条 `{ id, group, label, icon?, keywords, action }`）：

```
导航
  → 仪表盘 / 账号管理 / 执行控制 / 执行结果 / 配置设置
账号
  → 批量导入账号 / 添加单个账号 / 导出全部账号
执行
  → 执行全部 / 重试失败 / 停止流水线
代理
  → 应用并启动代理 / 检测出口 IP / 停止代理
配置
  → 打开协议模式开关 / 切换 link 来源到 API|Discord
主题
  → 切换暗色 / 切换亮色
```

- 模糊匹配：把 query 拆字符 case-insensitive 在 label + keywords 字段
  里 substring 匹配；按匹配位置 + group 优先级排序。
- 上下箭头 / 回车选中执行；Esc 关闭。
- 点击项也可执行。
- 命令注册抽象：`registerCommand({ id, group, label, action })`，供各
  视图 mount 时注册自己的命令（如 Execute 注册"执行选中"动态命令）；
  unmount 时反注册。

### 3.4 B4 Pipeline HUD

`web/src/components/ui/PipelineHUD.vue`：

- 挂在 AppLayout 主区顶部（紧贴顶栏下方），`v-if="execStore.running"`。
- 一条 60px 高的横条卡片：
  - 左：旋转的 spinner + "执行中"
  - 中：当前账号 email · phase · 已用时长 `02:14`
  - 右：进度 `12 / 50` + ETA `约 6:30 剩余` + 主代理节点 `pro-美国03`
  - 最右：跳转 [查看 Execute] [停止]
- 全局可见 —— 运营在 Accounts / Config 页也能看到 pipeline 在跑。
- 数据源：socket `account-status` 事件聚合 + 启动时间。

新增 `web/src/stores/pipelineStore.js`（普通 reactive 模块，非 pinia）：
```js
export const pipelineStore = reactive({
  running: false,
  total: 0,
  done: 0,
  currentEmail: '',
  currentPhase: '',
  startedAt: 0,
  recentDurations: [],     // 最近 5 个账号耗时，用于 ETA
})
```

socket.js 在 `execution-complete` / `account-status` 等事件里维护这个
store；Execute.vue 启动时设 startedAt + total。ETA = `平均最近 5 个
耗时 × (total - done)`。

## 附录 A：决策记录

- **不替换 el-tag 为 StatusBadge**：状态 tag 在 5+ 个视图被消费，全
  量替换风险高且没用户感知收益。v2.34 token 化阴影 / 圆角已让 tag
  视觉提升。留 v3.0。
- **Command Palette 而不是顶部菜单栏**：用户是技术运营，键盘党，
  Ctrl+K 比鼠标菜单效率高一倍；同时不撤侧栏（新手友好）。
- **Pipeline HUD 全局可见**：运营常在跑流水线时去 Config 改下次
  设置；HUD 全局显示让他不必切回 Execute 看进度。
- **ContextActionBar 而不是 toolbar disabled 按钮**：v2.34 toolbar
  第 2 行始终显示但选中=0 时按钮 disabled —— 视觉占地但无用。slide-
  in 上下文栏只在需要时出现。
- **任务卡片是按钮而不是统计**：Dashboard 不只是看，是"今天要做什
  么"的入口。
