# v2.34 — 前端重设计（设计 spec）

> 日期：2026-05-25  
> 范围：**保留所有现有功能 / API / 业务逻辑**。重做视觉系统、信息架构、
> 页面壳与一致性。不动后端、不动 status.js 单一来源、不动 socket /
> selection / engine。  
> 目标：简约大方（design tokens + 一致间距/字号）、符合人类操作
> （顶部全局头 + 清晰页面层级 + 标准空/加载/错误态）、功能全面
> （任何 v2.28~v2.33 已有功能不丢）。

## 一、为什么要做

v2.28~v2.33 在功能性上已经成熟（28 项优化、174→182 测试），但视觉与
信息架构跟不上：

- **配色硬编码**：AppLayout 侧栏 `#304156` navy 写死；Dashboard 卡片
  字号 `font-size:32px` 内联；多处直接写 `#909399 / #c0c4cc / #f56c6c`。
  暗色模式只是 Element Plus 自带变量切换，自定义颜色没跟着切。
- **缺顶部全局头**：通知/暗色/Socket 状态全塞到 logo 区，挤；五个视图
  顶部都没有"我在哪一页"的统一标题。
- **缺设计 token 层**：间距 / 字号 / 圆角 / 阴影都靠每个视图各自决定，
  视觉上不齐。
- **卡片低密度**：Dashboard 4+3 大卡占满屏，每张就一个数字。
- **Accounts toolbar 4 行**：横向元素过多，运营常找不到按钮。
- **空 / 加载 / 错误态不统一**：每个视图各自处理或不处理。

不做完整推倒重构（业务逻辑大量在视图里，迁移风险高）。**只重做：
设计 token / 全局 layout / 通用组件 / 5 视图按统一 IA 重排**。

## 二、不在范围内（明确排除）

- 后端、API、Socket 协议、status.js 状态码体系、selection.js 跨视图选中。
- 业务功能逻辑（添加/编辑/删除/导入/测活/支付/代理操作）按钮事件不动。
- 新增/替换前端框架；继续用 Vue 3 + Element Plus + Vite，**不引新 npm
  依赖**（图标用已有 `@element-plus/icons-vue`；色板用 Element Plus
  CSS 变量 + 项目自己一层）。
- 国际化（i18n）。
- 移动端响应式专项（仅做 `min-width: 1280px` 桌面优化；窄屏不撤已有的
  el-row gutter）。
- 单元测试新增（前端无测试基础设施；不在本轮引入）。

## 三、设计原则

1. **简约**：减少视觉噪音 — 去掉过粗边框、过深阴影、过多色块、过大
   字号；卡片间距用 16/24/32 三档；色板限定 7 个语义色 + 2 个 surface。
2. **大方**：留白比内容多；页面最大宽度 1440px 居中；左侧栏 220px /
   主区有舒展的 40px 横向内边距；字号缩到 13/14/16/20/24 五档。
3. **人性化**：每页有 PageHeader（标题 + 一句话副标题 + 主要操作槽）；
   危险按钮始终在右侧分组、确认走 `confirmDanger`；空 / 加载 / 错误态
   用统一组件。
4. **功能全面**：所有 v2.28~v2.33 已有功能 / 按钮 / 快捷键保留；只改
   摆放、视觉、层级。

## 六、通用组件（新增 3 个）

放在 `web/src/components/ui/`：

### 6.1 `PageHeader.vue`

```vue
<template>
  <header class="page-header">
    <div class="page-header__main">
      <h1 class="page-header__title">{{ title }}</h1>
      <p v-if="subtitle" class="page-header__subtitle">{{ subtitle }}</p>
    </div>
    <div class="page-header__actions"><slot name="actions" /></div>
  </header>
</template>
```

Props：`title` (string)，`subtitle` (string?)。slot `actions` 放右侧
按钮组。下边距 `--sp-5`。

### 6.2 `SectionCard.vue`

替代散落的 `<el-card>` 用法。

```vue
<template>
  <section class="sec-card">
    <header v-if="$slots.title || title" class="sec-card__head">
      <h3 class="sec-card__title">
        <slot name="title">{{ title }}</slot>
      </h3>
      <div v-if="$slots.extra" class="sec-card__extra"><slot name="extra" /></div>
    </header>
    <div class="sec-card__body" :class="{ 'sec-card__body--flush': flush }">
      <slot />
    </div>
  </section>
</template>
```

Props：`title` (string?)，`flush` (bool? 去掉 body padding，用于内嵌
表格)。背景 `--app-surface`、圆角 `--rad-md`、阴影 `--sh-1`、边框
`--app-border-soft`。

### 6.3 `EmptyState.vue`

```vue
<template>
  <div class="empty">
    <el-icon class="empty__icon"><slot name="icon"><DataLine /></slot></el-icon>
    <p class="empty__title">{{ title }}</p>
    <p v-if="hint" class="empty__hint">{{ hint }}</p>
    <slot name="action" />
  </div>
</template>
```

Props：`title` (string)，`hint` (string?)。slot `action` 放主按钮。

## 十三、风险与回滚

- **风险 1**：tokens.css 覆盖 Element Plus 变量过狠导致部分组件错位。
  缓解：tokens 用 `--app-*` 自命名空间，不直接覆盖 `--el-*`；仅在
  需要时通过 token 派生（如 `--el-bg-color: var(--app-surface)`）。
- **风险 2**：AppLayout 重做让通知 drawer / 断线 alert 行为变化。
  缓解：保持组件本身不动，只搬位置；socket / notifications 模块
  不动。
- **风险 3**：Config 左 tab rail 与现有 dirty form 守卫（v2.29 FX-6）
  交互可能错位。缓解：只换 tab-position 属性，不改 form watcher。
- **风险 4**：视觉重做后 e2e 用户对动线不适应。缓解：保留所有按钮
  文案 / 位置语义不变，只改样式；commit message 详细记录"按钮 X
  从 toolbar row N 移到 PageHeader actions"。

每 batch 一个 commit；任何 batch 出红立即 `git revert HEAD`，spec
对应小节修正后重做。

## 附录 A：决策记录

- **为什么不换组件库（如 Naive UI / Ant Design Vue / shadcn）**：
  Element Plus 已被项目深度使用（el-table reserve-selection / 通知 /
  drawer / tag / popover），换组件库 = 整套 UI 改写，2-3 天工作量
  且会丢现有 UX 细节（v2.28 行高亮 / 跨表选中等）。token 系统能在
  Element Plus 之上实现"简约大方"目标。
- **为什么 sidebar 不收成 icon-only 64px**：5 个视图都是高频项，
  没有侧栏空间真要让出去；运营每天看的是中文标签更直观。
- **为什么 Config 改左 tab rail**：当前 7 tab 横排在桌面 1366px 屏会
  挤；左 rail 既给 tab label 更多空间又把右侧整个表单视野留给字段。
- **为什么所有视图加 PageHeader**：解决"我在哪一页 / 这页能干什么"
  的视觉认知成本。这是简约大方与人性化的最低要求。
