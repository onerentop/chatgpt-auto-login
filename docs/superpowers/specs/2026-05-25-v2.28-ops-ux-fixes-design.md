# v2.28 Ops UX Fixes — 10 个 P1 改进打包

**Date:** 2026-05-25
**Status:** Draft → 待评审
**Predecessors:** v2.27.0 (execute status groups), v2.26.0 (account liveness), v2.21.0 (main proxy whitelist), v2.20.0 (proxy blacklist)

## Background

v2.18-v2.32 快速迭代积攒的债集中在 3 类，三模块审查（执行控制 / 账号管理 / 代理）共发现 32 个改进项，其中 10 个标记为 P1（强烈建议优先修）。本 spec 将这 10 项打包为 v2.28 一次性 ship。

3 类债：
1. **状态维度扩展未同步到 UI**：`status.js` LABEL_MAP 演化加了 `paypal_captcha / login_fail / token_expired / aborted` 4 个状态，但 Execute.vue 的 statusFilter 下拉硬编码 11 项没跟上；`failedEmails` 只识别 `error` 一个，"重试失败"按钮严重失真。
2. **后端能力 > 前端暴露**：DB 已有 `reason` / `updatedAt` 字段、proxyMgr 有 currentNode / exitIp 实时状态，但前端只显示 status tag。
3. **危险操作无守卫 + UI 体力活**：测活全部、停止代理无二次确认；Accounts 16+ 控件挤一行；Config 482 行平铺。

## Goals

10 个 P1 改进一次性 ship：

1. Execute `failedEmails` 改用所有可重试终态（9 个）替代单一 `error`
2. Execute statusFilter 下拉从 `status.js` 动态生成
3. Execute 子表加 `原因` 列 + expand-row 顶部 banner 显示代理节点/出口 IP/时间
4. Execute toolbar 显示协议模式 / payment link source badge
5. Execute Stop 按钮可用性靠 5s polling `/execute/status` 兜底
6. Accounts toolbar 拆 4 行（数据 / 筛选 / 测活 / 下载）
7. Accounts TOTP / Refresh Token / Client ID 列加 tooltip + 复制按钮
8. Accounts "测活全部"按钮加 ElMessageBox 二次确认
9. Config 整页用 7 个 el-tabs 重构（支付 / 执行 / Discord / OAuth / 代理 / JP / 黑名单）
10. Config "停止代理"按钮加 ElMessageBox 二次确认

## Non-Goals

- 不抽通用 composable（如 `useAuthDownload`）—— P3 范围
- 不改后端 API 契约，仅扩展 `account_status` 表 2 列 + `/api/results` 透传字段
- 不实现 P2/P3 共 22 项改进（留给后续 v2.29+）
- 不重写 form 校验（仍依赖服务端校验）

## 决策记录

| 决策点 | 选择 | 理由 |
|---|---|---|
| #1 重试范围 | 所有未拿到 Plus 的终态 9 个 | 运维最常需求"一键重跑所有没拿到 Plus 的"；deactivated 排除（账号已死） |
| #3 子表新列 | 加 `reason` 一列；代理节点/出口 IP 放到 expand-row 顶部 banner | reason 是高频查询字段值得上列；代理上下文低频但重要，banner 显示同时减少列宽压力 |
| #5 Stop 机制 | 加 5s polling `/execute/status` | socket 仍是 fast path；polling 仅作为冗余，断 socket 仍可 stop |
| #9 Config 导航 | 顶部 el-tabs 7 节 | UX 最清晰；用户进 Config 是"改某字段"语义，tab 比 collapse/anchor 更贴 |
| #3 后端配合 | 加 schema migration + engine 注入 | reason 已存但 proxy_node/exit_ip 不存；加 2 列后历史完整可查；后端 ~10 行 |
| v2.28 范围 | 全栈（前端 + 5 处后端文件） | #3 强依赖后端 schema；不拆 stretch 为两版本，一次 ship 减心智负担 |

## Architecture

```
后端基础设施层：
  server/db.js                          ← account_status 加 proxy_node/exit_ip 列 + PRAGMA-gated ALTER
                                          + statusDB.set merge-aware（如未实现）
  server/engine.js + protocol-engine.js ← emitStatus 读 proxyMgr.getState() 注入 proxy_node/exit_ip
  server/routes/results.js              ← 透传 proxyNode/exitIp 到响应

前端工具层：
  web/src/status.js                     ← FAILED_TO_RETRY_STATUSES + isFailedToRetry +
                                          EXECUTE_STATUS_FILTER_OPTIONS 三个新导出

前端模块层：
  web/src/views/Execute.vue             ← #1 failedEmails / #2 statusFilter 动态 /
                                          #4 mode badge / #5 polling
  web/src/components/AccountTableRows.vue ← #3 reason 列 + expand-row banner
  web/src/views/Accounts.vue            ← #6 toolbar 4 行 / #7 token tooltip+复制 / #8 二次确认
  web/src/views/Config.vue              ← #9 el-tabs 7 节 / #10 停止代理二次确认

文档：
  docs/CHANGELOG.md                     ← 追加 v2.28 节
```

**不动**：选择性测试基础设施 / 后端 API 契约 / proxy/blacklist.js / engines 业务逻辑 / liveness 模块 / selection.js。

### 数据流（#3 端到端）

```
engine emitStatus
  → proxyMgr.getState() 读 currentNode + exitIp
  → statusDB.set({ ..., proxy_node, exit_ip })   ← merge-aware: 缺失字段不覆盖
  → DB 写表
  → /api/results 响应含 proxyNode + exitIp + reason
  → Execute.vue loadResults 写入 row._proxyNode / _exitIp / _reason
  → AccountTableRows.vue 模板渲染 reason 列 + expand-row banner
```

### 关键不变式

1. **socket 仍是 fast path**：polling (#5) 是冗余路径，socket 推 `Execution complete` 时 watch 改 `running.value=false` 不变。polling 仅在 socket 断线时兜底。
2. **statusDB.set merge-aware**：与 v2.26 `payment_link*` 同模式 —— 缺失字段不覆盖既有值。proxy_node/exit_ip 中间态 emitStatus 不要清空已写入的值。
3. **后端 schema 迁移幂等**：PRAGMA-gated ALTER 确保老库升级、新装 / 重复运行都不抛 duplicate column 错。
4. **`status.js` 单一来源**：FAILED_TO_RETRY_STATUSES / EXECUTE_STATUS_FILTER_OPTIONS 是新加常量，与 PLUS_STATUSES / ERROR_STATUSES 平起平坐；任何"哪些状态算..."的判定都从这里读，不再在视图层硬编码。

## Data Structures

### `status.js` 增量

```js
// === Execute page retry / filter helpers ===

// 所有 "未拿到 Plus 但非账号死亡" 的终态 — "重试失败" 按钮的目标集
// 排除 deactivated（账号已删）、plus / plus_no_rt（已成功）、
// idle / running（未到终态）、checking / unknown（liveness 中间态）
export const FAILED_TO_RETRY_STATUSES = [
  'error',
  'no_link',
  'no_promo',
  'verify_error',
  'paypal_captcha',
  'login_fail',
  'token_expired',
  'aborted',
  'no_jp_proxy',
]

export function isFailedToRetry(status) {
  return FAILED_TO_RETRY_STATUSES.includes((status || '').toLowerCase())
}

// statusFilter 下拉的动态选项 — 从 LABEL_MAP 全集生成，避免下拉与状态体系脱节
// 排除纯 liveness 维度（checking / canceled），它们不在执行流水线 status
export const EXECUTE_STATUS_FILTER_OPTIONS = Object.entries(LABEL_MAP)
  .filter(([k]) => !['checking', 'canceled'].includes(k))
  .map(([value, label]) => ({ value, label }))
```

**为什么不复用 `ERROR_STATUSES`**：现有 `ERROR_STATUSES = ['error', 'no_link', 'deactivated', 'no_promo']` 含 deactivated 但缺 5 个状态。语义也不同 —— "重试"≠"错误"。

### `account_status` schema 增量

```sql
-- CREATE TABLE 块同步加（首次 install）
CREATE TABLE IF NOT EXISTS account_status (
  email TEXT PRIMARY KEY,
  status TEXT DEFAULT 'idle',
  phase TEXT DEFAULT '',
  progress TEXT DEFAULT '',
  reason TEXT DEFAULT '',
  has_auth_file INTEGER DEFAULT 0,
  updated_at TEXT DEFAULT (datetime('now')),
  proxy_node TEXT DEFAULT '',     -- 新增
  exit_ip TEXT DEFAULT '',         -- 新增
  alive_status TEXT DEFAULT 'unknown',
  alive_checked_at TEXT DEFAULT '',
  alive_reason TEXT DEFAULT ''
);
```

**老库迁移**（PRAGMA-gated ALTER，幂等）：

```js
const colsResult = db.exec("PRAGMA table_info(account_status)");
const colNames = new Set((colsResult[0]?.values || []).map(v => v[1]));
if (!colNames.has('proxy_node')) {
  db.run("ALTER TABLE account_status ADD COLUMN proxy_node TEXT DEFAULT ''");
}
if (!colNames.has('exit_ip')) {
  db.run("ALTER TABLE account_status ADD COLUMN exit_ip TEXT DEFAULT ''");
}
```

### #5 5s polling /execute/status

```js
let statusPollTimer = null

function startPolling() {
  const tick = async () => {
    if (document.visibilityState !== 'visible') return
    try {
      const { data } = await api.get('/execute/status')
      running.value = data.status === 'running'
    } catch {}
  }
  statusPollTimer = setInterval(tick, 5000)
  document.addEventListener('visibilitychange', tick)
}

onBeforeUnmount(() => {
  if (statusPollTimer) clearInterval(statusPollTimer)
})
```

### #3 loadResults 透传新字段

```js
async function loadResults() {
  try {
    const { data } = await api.get('/results')
    for (const r of data) {
      const row = accounts.value.find(a => a.email === r.email)
      if (row) {
        if (r.status && r.status !== 'idle') row._status = r.status
        if (r.phase) row._phase = r.phase
        row._hasAuth = r.hasAuthFile || false
        row._reason = r.reason || ''             // 新增
        row._proxyNode = r.proxyNode || ''       // 新增
        row._exitIp = r.exitIp || ''             // 新增
        row._updatedAt = r.updatedAt || ''       // 新增
        const st = (r.status || '').toLowerCase()
        row._plan = PLUS_STATUSES.includes(st) ? 'plus' : (['error', 'no_link'].includes(st) ? 'free' : '')
      }
    }
  } catch {}
}
```

## 风险与缓解

| # | 风险 | 缓解 |
|---|---|---|
| R1 | `engineMode` 加载时 GET `/config/raw` 失败 → toolbar badge 显示错误 mode | catch 兜底默认 `{protocolMode: false, paymentLinkSource: 'api'}`，与 server.js 默认一致 |
